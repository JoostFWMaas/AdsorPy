# Security Policy

## Supported Versions

Security updates are provided only for the latest released version of this project.

---

## Security Considerations

AdsorPy is a scientific Python library intended for research and educational use.
It does **not** handle authentication, authorisation, personal data, or network‑exposed services.

As such, it is **not designed to operate in security‑critical or adversarial environments**.

---

## Reporting a Vulnerability

If you believe you have found a security‑related issue, please report it **privately** by emailing:

[j.f.w.maas@tue.nl](mailto:j.f.w.maas@tue.nl)

PGP encryption is not currently provided. Plain‑text email is acceptable.

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)/a minimal working example (MWE)
- Any relevant logs or traces

---

## Non‑Security Issues

The following are **not considered security issues** and should be reported via normal GitHub issues:

- Bugs
- Performance problems
- Documentation errors
- Dependency updates
- Test failures

---

## Disclosure Policy

Please do not publicly disclose potential security issues until they have been reviewed.
We make no guarantees regarding response time or remediation, but we appreciate responsible disclosure.
